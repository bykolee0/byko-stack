# 질문 라우팅 상세

## 왜 질문 라우팅이 필요한가

AI가 스펙을 작성할 때 가장 흔한 실패 모드는 "모호한데 자체적으로 결정해버리는 것"이다.
특히 현재 컨텍스트에 있는 정보만으로 확신하여 국소적인 스펙을 만들거나,
정책적 결정(비즈니스 규칙, 허용 동작 등)을 AI가 임의로 채우는 문제가 반복된다.

질문 라우팅은 이 문제를 구조적으로 방지한다: 불확실한 항목을 만나면 먼저 "이게 어떤 유형의 불확실성인가?"를 분류하고, 유형에 따라 정해진 행동을 취한다.

## 4가지 유형

### Type A: 코드에서 확인 가능한 사실

코드를 읽으면 답이 나오는 항목. AI가 먼저 확인하고 유저에게는 confirmation만 구한다.

**판별**: "이 질문의 답이 코드 어딘가에 있는가?"
**행동**: 코드를 읽고 확인 → 유저에게 "코드를 보니 X인데, 맞습니까?" 식으로 확인
**ledger**: `from-code` (파일:라인 근거 필수)

**예시:**
- "현재 인증 방식이 뭔가요?" → 코드에서 미들웨어 확인 가능 → Type A
- "기존 API 응답 포맷이 어떤가요?" → 코드에서 확인 가능 → Type A
- "테스트 프레임워크가 뭔가요?" → package.json/pyproject.toml에서 확인 → Type A

### Type B: 유저만 결정할 수 있는 사항

제품 방향, 비즈니스 정책, 우선순위, 허용 동작 등 코드에 답이 없는 항목.
**반드시 유저에게 질문한다. 절대 AI가 자체 결정하지 않는다.**

**판별**: "이 결정은 코드가 아니라 사람(PM, 개발자, 이해관계자)이 내려야 하는가?"
**행동**: 유저에게 직접 질문
**ledger**: 유저 답변 전까지 `open` 또는 `blocking`, 답변 후 `confirmed`

**예시:**
- "삭제는 soft delete? hard delete?" → 비즈니스 정책 → Type B
- "이 기능의 대상 사용자는?" → 제품 결정 → Type B
- "에러 시 재시도 정책은?" → 정책 결정 → Type B
- "무료 플랜에서도 이 기능을 제공하나요?" → 비즈니스 → Type B

### Type C: 코드 사실 + 정책 판단 혼합

코드에서 현재 상태는 확인할 수 있지만, 앞으로의 방향은 유저 판단이 필요한 항목.

**판별**: "코드로 '현재 어떻게'는 알 수 있지만, '앞으로 어떻게'는 결정이 필요한가?"
**행동**: 코드 사실을 먼저 정리한 후, 판단 부분만 유저에게 질문
**ledger**: 코드 사실은 `from-code`, 판단 부분은 유저 답변 후 `confirmed`

**예시:**
- "현재 rate limit이 100req/min인데(from-code), 이 새 API도 같은 기준을 적용할까요?(유저 결정)"
- "기존 에러 응답이 { error: string } 포맷인데(from-code), 이 API도 같은 포맷으로 갈까요?(유저 결정)"

### Type D: 판단이 필요한데 정보가 없음

코드에도 없고, 유저에게 아직 물어보지도 않은 상태.
**절대 자체 생성 금지.** 반드시 `blocking`으로 등록하고 유저에게 질문한다.

**판별**: "이 항목에 대해 아무런 근거가 없는데, 내가 임의로 채우려 하고 있는가?"
**행동**: ledger에 `blocking`으로 등록, 유저에게 질문
**ledger**: `blocking` → 유저 답변 후 `confirmed`

**예시:**
- 데이터 마이그레이션이 필요한지 모르는데 "필요 없다"고 가정하려는 경우 → Type D, blocking
- 에러 응답 포맷에 대한 기준이 없는데 임의로 정하려는 경우 → Type D, blocking

## 자기 점검

질문 라우팅을 제대로 적용하고 있는지 확인하는 체크리스트:

- [ ] ledger에 `assumed` 항목이 있다면, 그 중 Type B나 D로 분류되어야 할 것이 섞여 있지 않은가?
- [ ] "이건 당연히 이렇겠지"라고 넘어간 가정이 있지 않은가?
- [ ] 코드에서 확인한 사실에 파일:라인 근거를 기재했는가?
- [ ] 유저에게 질문할 때 Type A(코드 사실)와 Type B(유저 판단)를 구분하여 전달하고 있는가?
